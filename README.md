# 🌾 AssurPrime — Agricultural Fire Insurance Pricing

> **ChallengeData** · Crédit Agricole Assurances  
> Predicting pure fire insurance premiums for agricultural multi-risk contracts

**Team:** Pierre-Antoine Becq, Charles Delbecque, Clara Gracieux, Louis Faure-Comte · *March 2026*

---

## 📋 Overview

This project models the **pure fire insurance premium** (`CHARGE`) for agricultural contracts managed by Pacifica, using the classic actuarial decomposition:

$$\text{CHARGE} = \text{FREQ} \times \text{CM} \times \text{ANNEE\\_ASSURANCE}$$

Two separate models are trained:
- **Frequency** (`FREQ`): claim occurrence rate → XGBoost (Poisson objective), trained on all policies
- **Severity** (`CM`): average cost per claim → XGBoost (Gamma objective), trained only on policies with at least one claim ( $\text{FREQ} > 0$ and $\text{CM} > 0$ )

Performance is evaluated via **RMSE on CHARGE**, benchmarked against a Ridge GLM baseline.

## 🔧 Pipeline

### Preprocessing (`src/features.py`)
- Bracket intervals (e.g. `[10k - 50k]`) and threshold strings (e.g. `<= 10`) are parsed into numerics via regex
- Missing values follow business logic: surface/capital → `0`, temporal → median, categorical → `MISSING` label
- Low-variance ( $< 0.01$ ) and highly correlated ( $> 0.95$ ) features are dropped
- Numerics: Robust scaling — Categoricals: Target Encoding (5-fold CV in final pipelines)

### Tuning (`src/tune.py`)
Optuna optimizes XGBoost hyperparameters separately for FREQ (Poisson) and CM (Gamma), with preprocessing applied outside the tuning loop for efficiency.

### Training (`src/train.py`)
Full 5-fold OOF cross-validation ensures models are never evaluated on data they were trained on. Final models are retrained on the complete dataset before generating test predictions.

## 📈 Results

5-fold Out-Of-Fold (OOF) cross-validation:

| Model | RMSE Frequency | RMSE Severity (CM) | RMSE Final Premium (CHARGE) |
|---|:---:|:---:|:---:|
| Baseline (Ridge) | 0.2918 | €99,079 | €7,087.19 |
| XGBoost (Optimized) | **0.2912** | **€78,944** | **€6,796.44** |

XGBoost reduces severity error by ~€20,000 and delivers an absolute CHARGE improvement of **€290.75 per policy**. The XGBoost model is retained for submission. In a production context, a distributed GLM (e.g. via Spark) would be worth considering for its regulatory explainability (ACPR) and lower compute cost.

## 🏭 Deployment

The production architecture rests on three pillars:

1. **Batch Processing** — premiums are recalculated asynchronously during annual renewal campaigns, not in real-time
2. **ETL Pipeline** — the preprocessing script is wrapped in an automated workflow to standardize raw enterprise data before inference
3. **REST API** — models are serialized via `joblib` and served as a Model-as-a-Service; advisors' tools query the API and receive `FREQ`, `CM`, and `CHARGE` without interacting with the model directly
